# THE POINT 一点 · 平台化改造技术方案

> 目标:房产中介网站 → 新西兰高端华人一站式生活门户平台
> 口号:落地新西兰,一点全通 · One Point, All Connected
> 技术:沿用 Node 22 + Express + node:sqlite + Multer,单文件夹部署不变

---

## 一、数据库

### 1. 新表

```
users            用户(统一账号,注册不分身份)
  id, email UNIQUE, password_hash, salt, name, phone, wechat, avatar,
  status('active'/'banned'), created_at

vendors          入驻方(申请与主页一体,status 驱动审核)
  id, user_id UNIQUE → users,
  name_zh, name_en, section_id, category_id,
  tagline_zh/en, intro_zh/en,
  phone, email, wechat, website, logo, photos(JSON),
  verified(一点认证), featured(推荐位),
  status('pending'/'approved'/'rejected'/'suspended'),
  apply_message, review_note, sort_order, created_at, approved_at

sections         板块(一级,后台可管理)
  id, slug UNIQUE, name_zh, name_en, tagline_zh, tagline_en,
  icon, sort_order, visible

categories       分类(二级,后台可管理)
  id, section_id → sections, slug, name_zh, name_en, sort_order, visible

posts            统一内容表(所有 UGC + 官方文化内容)
  id, user_id, vendor_id(空=普通用户发布), section_id, category_id,
  type: market(二手) / service(服务信息) / update(动态) / event(活动)
        / job(招聘) / seek(求职) / article(文章) / video(视频),
  title_zh/en, content_zh/en, images(JSON), video_url,
  price_label, price_value, contact_name/phone/wechat/email,
  location, event_time, extra(JSON),
  status('pending'/'approved'/'rejected'/'offline'),
  featured(编辑推荐=主页精选流), review_note, created_at, published_at
```

### 2. 老表迁移(启动时 PRAGMA 检查自动补列)

- properties:+ title_en, description_en
- news:+ title_en, excerpt_en, content_en
- agents:+ bio_en
- services:+ tagline_en, description_en

### 3. 初始板块与分类(种子,后台可增删)

| 板块 slug | 中/EN | 初始分类 |
|---|---|---|
| property | 置业 Property | 买房/卖房/租赁/新盘/置业专家(公司自营,复用现有房源功能) |
| advisory | 顾问 Advisory | 税务/会计/律师/移民/贷款/保险/规划师/测量师 |
| education | 教育 Education | 留学/私校咨询/学科私教/语言/才艺/体育教练 |
| living | 生活 Living | 家政/除草园艺/维修装修/搬家/餐饮/健康/美容/宠物 |
| auto | 车辆 Auto | 二手车买卖/租车/维修保养 |
| market | 集市 Market | 家具/电器/母婴/其他(普通用户可发) |
| local | 同城 Local | 活动/城中资讯(入驻方也可发活动) |
| career | 招聘 Jobs(原「机遇」,已更名) | 企业招聘/个人求职 |
| culture | 文化 Culture | 文章/视频(官方主导,支持外部视频嵌入) |

### 4. 权限模型

- 游客:浏览全部已审核内容
- 普通用户:发布 type=market(集市二手);申请入驻
- 入驻方(vendor approved):自建主页 + 发布 service/update/event/job
- 管理员:审核一切;官方发布 article/video;认证/推荐位
- **一切公开内容 status='approved' 才展示**;用户编辑已发内容 → 重置 pending 重审

会话:用户与管理员各一个内存 Map(cookie `tp_uid` / `tp_sid`),重启失效,前期够用。

## 二、API 路由

### 用户认证 `/api/auth/*`
- POST /api/auth/register {email,password,name,phone}
- POST /api/auth/login | POST /api/auth/logout
- GET /api/auth/me(含 vendor 状态)
- PUT /api/auth/profile | POST /api/auth/change-password
- POST /api/user/upload(登录用户传图,≤6 张/次、≤8MB)

### 入驻方自助 `/api/vendor/*`
- POST /api/vendor/apply(创建/重新提交,→pending)
- GET /api/vendor/me
- PUT /api/vendor/me(改资料 → 重置 pending 重审)

### 我的发布 `/api/my/posts`
- GET / POST / PUT /:id / DELETE /:id(POST 按身份校验可发 type;编辑→pending)

### 公开 `/api/*`
- GET /api/sections(含 categories,仅 visible)
- GET /api/posts?section=&category=&type=&q=&page=(仅 approved,featured 排前)
- GET /api/posts/:id
- GET /api/vendors?section=&category=&q=(仅 approved;featured、verified 排前)
- GET /api/vendors/:id(含其已审核 posts)
- GET /api/home(主页聚合:板块+精选内容流+精选入驻方+精选房源+最新资讯)

### 后台新增 `/api/admin/*`(requireAuth)
- users:GET 列表 / PUT :id(封禁/解封)/ DELETE
- vendors:GET ?status= / PUT :id(approve/reject/suspend/verify/feature/编辑)/ DELETE
- posts:GET ?status= / POST(官方发布,直接 approved)/ PUT :id(approve/reject/feature/编辑)/ DELETE
- sections、categories:纳入现有 TABLES 通用 CRUD
- stats 扩展:待审入驻、待审内容、用户数

## 六、打磨批次(第二轮优化,已完成)

### 新增表
```
sessions        会话持久化(token, kind: user/admin, ref_id, expires)
                → 重启不掉线;用户 30 天滑动续期,管理员 12 小时;启动+每 6h 清理过期
notifications   站内通知(user_id, type, title/body 双语, link, is_read)
settings        键值设置(site_url, smtp_* 邮件配置)
enquiries       + vendor_id 列(给入驻方留言)
```

### 通知触发点(server.js `notify()`)
注册欢迎 / 入驻批准·驳回·暂停 / 授予认证 / 内容通过·驳回·下架 / 收到留言。
前台:导航铃铛红点(i18n.js)+ 个人中心通知列表(打开即标记已读)。
邮件:`sendMail()` 为预留接口,后台「设置」配好 SMTP 后接 nodemailer 即可。

### 新 API
- GET /api/search?q= —— 双语跨库搜索(房源/入驻方/内容/资讯),前台 search.html
- GET /api/my/notifications、PUT …/read-all、PUT …/:id/read
- GET /api/my/enquiries(入驻方收到的留言)
- GET/PUT /api/admin/settings
- GET /sitemap.xml(数据库实时生成)、GET /robots.txt

### 内容防护
`lockContacts()`:未登录访客在 posts/vendors/home/search 响应中拿不到电话/微信/邮箱,
返回 `contact_locked:1`,前台渲染「登录查看」框(带回跳 back 参数)。

### 体验与性能
- 骨架屏(skel)、空状态(empty-state)、上传即时预览(fprev)+ canvas 压缩(≤1600px JPEG 0.82)
- 表单校验强化(邮箱格式服务端校验)、后台 toast 操作反馈、404.html
- 静态资源 Cache-Control 7d / 上传 30d / HTML no-cache;列表 API 剔除 extra/review_note、正文截断 200 字
- 全页双语 title/description/OG(微信卡片同用);详情页 JS 动态改写 og:title/description/image
- 移动端:导航 1460px 隐藏英文副标、1130px 收进抽屉;个人中心/表单/搜索单列适配;prefers-reduced-motion

### 回归结论
API 13 项 + jsdom DOM 级 24 项(含登录态、锁定、通知红点、EN 模式)全部通过。

## 七、管理员管控批次(第三轮,已完成)

### 新表
```
admin_logs   操作日志(admin, action, target_type/id/label, detail, created_at)
```

### 用户管控(/api/admin/users*)
- 列表支持 ?q=关键词&status=&from=&to=&sort=(邮箱/称呼/电话搜索,状态与注册时间筛选)
- GET :id 详情 = 资料 + 入驻状态 + 全部发布记录 + 在线会话数
- 封禁:立即删除其全部会话(踢下线)+ 无法登录;**公开内容与入驻主页通过可见性联动自动下架**
  (公开查询一律 JOIN users 校验 status='active',不改内容状态 → 解封自动恢复,零副作用)
- PUT :id/reset-password:重置密码(可自动生成,仅返回一次)+ 踢下线
- POST :id/offline-posts:一键下架其全部内容
- DELETE:注销,级联删发布/入驻主页/会话/通知,前台二次确认

### 入驻方 / 内容
- 入驻:暂停→「恢复入驻」、新增「取消资格」(→rejected+原因通知);内容:下架/删除/编辑已有
- 内容审核新增类型(二手/服务/活动/文章/视频…)与板块筛选

### 操作留痕
封禁/解封/注销/重置密码/一键下架/入驻批准驳回暂停认证推荐/内容过审驳回下架删除,共 19 类动作
全部写入 admin_logs;后台「操作日志」tab 可按操作人/对象/动作检索(最近 300 条)。

### 前台↔后台对照结论
门户推荐位(精选流/优选入驻方/臻选房源)、九板块与分类、房源、专家、资讯、伙伴、服务名录、
各类帖子、文化文章视频、留言、用户——后台均有管理入口,可看/可改/可控显示(精选、排序、上下架)。
唯主页品牌文案与 hero 图属代码层静态设计,不入库。

### 管控链路自测(全过)
封禁→旧会话 401 踢下线→登录被拒→帖子/主页/首页精选/搜索全部消失→解封全部自动恢复;
重置密码旧密码失效+踢下线;一键下架;注销级联清空;19 类操作日志逐条可查。
